Docs/Introduction

Apocalypse Hub API

The Apocalypse Hub HTTP API lets script developers manage keys, users, and scripts programmatically — and powers the on-the-fly obfuscated script delivery used by executors.

v3REST / JSONBearer auth60 req/min
⛔

This is not a public API. Endpoints are available to Apocalypse Hub customers and verified script developers only. You must whitelist your server's IP address on the dashboard before making calls — otherwise requests are blocked at the edge.

Base URL

All API requests are made to:

https://api.apocalypsehub.com

Every request must include a Content-Type: application/json header, and (unless noted) an Authorization header containing your API key.

Core concepts

  • Keys follow the format APOC-XXXX-XXXX-XXXX and are either free (12-hour, gated) or paid (30-day, account-bound).
  • Projects group scripts. A user whitelisted in a project can access every script within it.
  • HWID auto-assignment — if a key has no HWID, it binds to the first machine that executes the script.
  • Paid keys require a matching uid parameter so a key only works for its owner's account.

Authentication
Getting Started/Authentication

Authentication

Most endpoints require an API key passed in the Authorization header. Your API key is tied to your account and grants access to your projects.

⚠️

Treat your API key like a password. Anyone with it can create, delete, and blacklist keys across your projects. Rotate it from the dashboard if it leaks.

Authorization header

Send your key in the Authorization header on every authenticated request:

request headers
Authorization: apoc_live_3f9a2c8e1b7d4a06e5c2
Content-Type: application/json

IP whitelisting

⛔

You must whitelist your server's IP under Dashboard → Developer → API Access. Requests from non-whitelisted IPs are rejected with 403 before reaching the API.

Test your credentials

cURL
curl https://api.apocalypsehub.com/v3/keys/YOUR_API_KEY/details \
  -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json"

Rate Limits & Errors
Getting Started/Rate Limits & Errors

Rate Limits & Errors

The API allows a maximum of 60 requests per minute per API key. Exceeding this returns a 429.

Status codes

CodeMeaning
200Success — the request completed.
400Bad request — malformed body or missing required parameter.
403Forbidden — invalid API key, or IP not whitelisted.
404Not found — the key, user, or script does not exist.
429Too many requests — you exceeded 60 req/min.

Error response shape

403 Forbidden
{
  "success": false,
  "message": "Invalid API key or IP not whitelisted.",
  "code": "UNAUTHORIZED"
}

Quick Start
Getting Started/Quick Start

Quick Start

Generate a key and deliver a protected script in three steps.

1. Create a user key

cURL
curl -X POST https://api.apocalypsehub.com/v3/projects/PROJECT_ID/users \
  -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "key_days": 30, "note": "customer#1024" }'

2. Share the loadstring

Lua
loadstring(game:HttpGet(
  "https://api.apocalypsehub.com/v3/scripts/serve/SCRIPT_ID?key=" .. key
))()

3. The user executes

On execution the API validates the key, obfuscates the Lua on-the-fly, and returns the protected payload. The HWID binds automatically on first run.

✅

That's it — your script is now key-locked and obfuscated. Manage the key's lifetime, HWID, and blacklist status through the endpoints in this reference.

Create Key
User & Key Management/Create Key

Create a Key / User

Every user you add gets a random APOC-XXXX-XXXX-XXXX key generated for them. Omit assignment fields to create an unassigned key a user can later claim.

POST/v3/projects/:project_id/users

Body parameters

ParameterDescription
key_days optionalnumber — days the key stays valid, counted from first use.
auth_expire optionalnumber — Unix timestamp for a fixed expiry (alternative to key_days).
identifier optionalstring — pre-bind the key to an HWID or user identifier.
discord_id optionalstring — link a Discord account to the key.
note optionalstring — internal note shown in the dashboard.
💡

If you don't pass identifier or discord_id, the key is created unassigned — the first user to execute claims it and auto-binds their HWID.

Example request

cURL
curl -X POST https://api.apocalypsehub.com/v3/projects/prj_a91f/users \
  -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "key_days": 30, "note": "sale-1024" }'

Responses

200 OK
{
  "success": true,
  "user_key": "APOC-7F2A-9C1B-04E5",
  "auth_expire": -1,
  "key_days": 30
}
400 Bad Request
{ "success": false, "message": "key_days must be a positive integer" }

Serve Script
Script Management/Serve Script

Serve a Script

The core delivery endpoint. Validates the key, obfuscates the Lua on-the-fly, and returns the protected payload for the executor to run.

GET/v3/scripts/serve/:script_id?key=KEY&uid=UID

Parameters

ParameterDescription
key requiredquery — the user's APOC-… key.
uid conditionalquery — required for paid keys; must match the key's owner.

Validation flow

  • Confirms the key exists and is active.
  • Paid key → requires uid, verifies it matches assigned_to, and checks the plan is still active.
  • Free key → checks the script_keys authorization table.
  • Obfuscates the script and returns it.

Executor usage

Lua
loadstring(game:HttpGet(
  "https://apocalypsehub.com/api/scripts/serve/SCRIPT_ID?key=KEY&uid=UID"
))()
⛔

Paid keys are account-bound. If uid is missing or doesn't match assigned_to, the API responds with an obfuscated error stub instead of the script.

Key System
Guides/Key System

The Free Key System

Free users unlock a 12-hour key by completing a link-shortener task (Work.ink or Linkvertise). This monetizes free access without a subscription.

Flow

  1. User visits /get-key.
  2. Clicks the Work.ink or Linkvertise link.
  3. Completes the task → redirected back with ?task=completed&provider=xxx.
  4. Frontend auto-claims the key via POST /api/keys.
  5. The key is linked to all published scripts via the script_keys table and expires in 12 hours.
⚠️

Always validate the completion callback server-side against the provider's token. Never trust the task=completed query flag alone — it's trivially spoofable.

Paid users

Scripter plan skips the gate entirely and generates an account-bound key at /dashboard/keys via POST /api/keys/paid.

Loadstring Usage

Distribute your script as a one-line loadstring. The executor fetches, and the API returns obfuscated Lua.

Free key

Lua
loadstring(game:HttpGet(
  "https://apocalypsehub.com/api/scripts/serve/SCRIPT_ID?key=KEY"
))()

Paid key (account-bound)

Lua

  "https://apocalypsehub.com/api/scripts/serve/SCRIPT_ID?key=KEY&uid=USER_ID"
))()
✅

The uid ties the key to the owner's account — the key only works when the two match.

Obfuscation Pipeline
Guides/Obfuscation Pipeline

Obfuscation Pipeline

Every served script passes through a four-stage Lua obfuscator before it leaves the server.

StageWhat it does
encodeStringsConverts string literals into character-code arrays.
addJunkCodeInserts dummy variables and dead branches.
renameVariablesRewrites locals, e.g. local x → _lIlI0.
wrapInExecutorWraps everything in a closed function guarded by a token.
⚠️

Obfuscation is friction, not security. The full script still reaches the executor, so a determined user can reverse it. Rely on key validation and HWID binding for access control — treat obfuscation as deterrence for casual copying.
